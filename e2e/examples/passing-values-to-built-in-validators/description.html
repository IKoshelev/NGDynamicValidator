Oftentimes you need to make a custom validator function and share it across you project. validators service available through injection allow you to do that. Actual extension of the service is demonstrated in the next example and this one concentrates on its built-in functions and different ways you can pass parameters to them.

Built-in validator functions are required, minLength, maxLength and pattern. They pretty much resemble their counterparts available with vanilla Angular, and will stop $veiwValue propagation towards model if it is invalid, returning undefined.

All of them accept a parameter and return the actual validator function tied to that parameter. It can be:

  • minLength(3) - inline value
  • minLength(value) - inline value from $scope $scope.value = 3; Do note, chainging $scope.value afterwards does not affect the already passed function in this case.
  • minLength(valueObj) - an object on the $scope, whose .value property will be eveluated every time that this validation function fires. $scope.valueObj = { value: 3 };
  • minLength(valueProviderFn) - a value provider function attached to the $scope, that will be called for actual value during each validation. $scope.valueProviderFn = function () { return $scope.valueObj.value;};
