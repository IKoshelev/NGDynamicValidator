NGDynamicValidator directive provides your direcitves with a contoller that allows the dynamically change most aspects of its behaviour.

You can request it in a standard Angular way in another directive 

            
require: "ngDynamicValidator",
and it will be injected into your link function.

The controller exposes more members then described here, but the ones we don't mention are the ones you shuld not use, or use at your own risk.

  • getOptionsProp(propName);
    setOptionsProp(propName, newValue);     - allows you to get or set a property. Inteded for use with "_excludedNonValids", "_validateTogetherGroupsBehaviour"
  • addToValidateTogetherGroup(groupName);
    removeFromValidateTogetherGroup(groupName);     - add or remove this input to a validateTogetherGroup
  • addRequestGroup(groupName);
    removeRequestGroup(groupName);     - add or remove this input to a requestGroup
  • addValidator(eventName, validator);
    removeValidator(eventName, validatorFn);
    removeValidatorByName(eventName, validatorFnName);     - add or remove a validator function to a particular event. Do note, that removeValidator matches functions as objects via ===, so you would have to keep the reference to the original function object to use it. Instead - use removeValidatorByName, which checks and removes all validator functions with a matching name. In case you will need a list of all events that this particular controller reacts to, you can get it via options.validators or via getOptionsProp('validators') - each key will be an event name with a value of validators array. But be careful with this object - as it is the one used for internal opertaion of NGDynamicValidator, don't modify it directly, just get the list of keys. Same goes for options object.
